# Recovery protocol

> The system fails. The question is how fast you recover. Five common failure modes; each has a defined exit ramp.

## Failure mode 1 — Sleep debt accumulating (3+ nights &lt;6h)

Day 1: cancel optional tonight. In bed by 21:30. 8.5h TIB target. No alcohol, no afternoon caffeine.

Day 2: 30-min outdoor light walk before 09:00. Same rules. No deep work demands >2h.

Day 3: resume normal. If still impaired, extend to Day 4.

Per [Belenky 2003](https://pubmed.ncbi.nlm.nih.gov/12603781/): weekend recovery is partial. You can't fully repay 5×6h with 1×10h.

## Failure mode 2 — Exercise lapse (5+ days)

20-min walk Day 1. 30-min Day 3. 40-min Day 5. Resume 3×/week Day 7.

Don't double the volume.

## Failure mode 3 — Deep work collapse (>50% drop vs baseline)

Audit where the hours went (open `/insights`, look at last 7d).
- Meetings? Cap at 2/day.
- Slack? Set notification hygiene.
- Job-switching? Pick ONE day per job per week.

Schedule ONE 90-min block tomorrow morning. Just hit start.

## Failure mode 4 — Multi-project overload (>2 active)

Force one project Active → Maintenance or Paused **today**. Communicate the pause to stakeholders. The lie of "I'll get to it" is the worst path.

## Failure mode 5 — Interview-prep crunch

4-week sprint, not 4-month grind. Pause maintenance projects. Down to 1 active job + 1 active prep. Sleep stays at 7h. Skip "one more LeetCode" — diminishing returns.

## Meta-rule — 3-strikes recovery

If 3 different recovery protocols fail in 4 weeks: stop adding things, strip projects to 1, write a decision-log entry, consider a real off-week.
