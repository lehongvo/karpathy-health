# Deep work protocol

> The output engine. Without it, the rest of the system is decoration.

## Anatomy of a block

```
T-15 min: shutdown all notifications, close all unrelated tabs
T-10 min: write the ONE question you'll answer in this block
T-0:     start timer (DeepWorkTimer in this app)
T+90:    timer ends. Stop. Don't push past unless you're in flow.
T+95:    log distraction count + subjective focus 1-5
T+5 min walk before next block
```

## Hard rules

- **Block length**: 90-120 min (BRAC-aligned).
- **Daily target**: ≥1 block, ideally before noon.
- **Notifications during the block**: all off (Slack, email, phone).
- **One project per block.** No tab-switching.
- **Communication batched 2× per day**, 30 min each (e.g., 11:30 and 16:30).

## The attention residue cost

[Sophie Leroy 2009](https://www.sciencedirect.com/science/article/abs/pii/S0749597809000399) — switching from Task A to Task B under time pressure leaves measurable cognitive residue. The popular "20-40% productivity loss" attributed to Leroy is **not** in the original paper; the rigorous claim is "measurable decrement under time pressure."

[Mark, Gudith, Klocke 2008](https://ics.uci.edu/~gmark/chi08-mark.pdf) — average ~23 min 15 s to resume the originally interrupted task in office field study. Not independently replicated with that exact figure; treat as order-of-magnitude.

## Founding fathers

- [Cal Newport — Deep Work](https://calnewport.com/talk-to-your-boss-about-deep-work/) and [time blocking](https://calnewport.com/deep-habits-the-importance-of-planning-every-minute-of-your-work-day/)
- [Paul Graham — Maker's Schedule, Manager's Schedule](https://paulgraham.com/makersschedule.html) — meetings as half-day-destroyers
- [Andrej Karpathy — PhD survival](http://karpathy.github.io/2016/09/07/phd/) — focus calculus

## What this protocol does NOT endorse

- **Pomodoro 25/5 as default** — too short for hard cognitive work. Useful only as a get-unstuck tool.
- **Continuous Slack monitoring** — pure context-switch tax.
- **"I'll just check this quickly"** — see Mark 2008.
- **Hybrid maker/manager days** — a 12:30 meeting on a maker day = no maker day.
