# Decision Log

Track non-obvious system pivots. Every entry: date, what changed, why, expected outcome. Re-read every quarterly review.

---

## Template

```
### [YYYY-MM-DD] — [Short title]

**What changed**: [the actual change]

**Why**: [the constraint or evidence that drove it]

**Expected outcome by**: [date] — [measurable outcome]

**Reviewed**: [YYYY-MM-DD] — [hit / partial / missed; what now]
```

## What goes here vs the weekly review

- **Weekly review** = stats + next-week tactical adjustments.
- **Decision log** = strategy/system-shape changes. Adding a new framework. Dropping a habit. Pivoting a project.

If you're adjusting a number (target sleep, target deep work hours), that's a weekly tweak, not a decision log entry.

If you're adjusting a *philosophy* (deciding to drop the deep work block in favor of pomodoro, deciding to triple your active project count), that's a decision log entry. Force yourself to write it down. Future-you will need the rationale.

---

## Anti-pattern: too many entries

If you write a decision-log entry every week, you're not running the system — you're tinkering with it. Tinkering with the system is procrastination on the actual work. Cap at ~1 entry per quarter.
